When we started this project, the goal wasn't just to make "another UI kit." It was to address the bloat and latency that has plagued desktop development for a decade.

## The Metrics that Matter

Benchmarks based on a standard "Hello World" application on macOS (M1/M2).

| Metric          | Gödel        | Electron      | Wails v2       |
| :-------------- | :----------- | :------------ | :------------- |
| **Binary Size** | ~5-12 MB     | 150+ MB       | 20-40 MB       |
| **Startup UI**  | < 100ms      | 1s - 3s       | 200ms - 500ms  |
| **Logic Latency**| < 1ms        | 5-20ms        | 10-50ms        |
| **Memory Idle** | ~15-25 MB    | 100+ MB       | 40-60 MB       |
| **CPU (Idle)**  | **0.0%**     | ~1.0% - 2.0% | ~0.5% - 1.5%  |

### Evaluation Summary

*   **Logic Latency**: Because we use direct Go calls without a JSON bridge, your business logic responds as fast as the hardware allows.
*   **Startup Speed**: No large JS engine or WebKit runtime to load. The binary is ready to paint pixels the moment you click the icon.
*   **Battery Life**: Our event-driven architecture ensures that if nothing is changing on the screen, the GPU and CPU stay completely idle.

<Tip>
  You can run your own local evaluation using the CLI command: `godel report`.
</Tip>
